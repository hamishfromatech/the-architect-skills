# Document Formatting Standards

## General Formatting Guidelines

### Typography
- **Font Families**: Arial, Calibri, Times New Roman, or similar professional fonts
- **Body Text**: 11-12pt for readability
- **Headings**: 14-16pt for major headings
- **Title**: 18-24pt, bold
- **Line Spacing**: 1.15 or single for body text
- **Margins**: 1 inch standard, minimum 0.5 inch

### Page Layout
- **Page Size**: US Letter (8.5" × 11") or A4
- **Orientation**: Portrait (standard), Landscape (tables/charts)
- **Headers**: Include document title or page number
- **Footers**: Include page numbers and date

---

## Document Types and Their Structure

### Proposals
1. **Title Page**: Project name, client, date, version
2. **Executive Summary**: 1-2 pages max
3. **Problem Statement**: Clear, concise
4. **Solution/Approach**: Detailed methodology
5. **Timeline**: Visual schedule
6. **Pricing**: Clear cost breakdown
7. **Terms & Conditions**: Legal framework
8. **Appendix**: Supporting materials

### Contracts
1. **Parties**: Clear identification of all parties
2. **Recitals**: Background and purpose
3. **Definitions**: Key terms defined
4. **Terms**: Core obligations
5. **Payment Terms**: Amount, schedule, method
6. **Duration**: Start and end dates
7. **Termination**: Conditions and procedures
8. **Signatures**: Dated signature blocks

### Memos
1. **Header**: To, From, Date, Subject
2. **Purpose**: Clear statement of intent
3. **Background**: Context (if needed)
4. **Main Content**: Details and discussion
5. **Action Required**: Clear call to action
6. **Distribution**: CC list

### Letters
1. **Letterhead**: Company information
2. **Date**: Current date
3. **Recipient Address**: Full address
4. **Salutation**: Appropriate greeting
5. **Body**: Clear paragraphs
6. **Closing**: Professional sign-off
7. **Signature**: Name and title
8. **Enclosures**: List of attachments

## Heading Hierarchy

```markdown
# Title (H1) - Document title only
## Major Section (H2) - Main sections
### Subsection (H3) - Subsections
#### Sub-subsection (H4) - Detailed breakdowns
```

### Rules
- Don't skip heading levels
- Maximum 4 levels of headings
- Keep heading text concise
- Use consistent formatting

---

## Lists and Bullet Points

### When to Use Bullets
- Listing items without order
- Short phrases (less than a sentence)
- Emphasizing key points

### When to Use Numbers
- Sequential steps
- Priority order
- Counted items

### Formatting
```markdown
- Bullet point text should be concise
- Parallel structure for all items
  - Nested bullets for sub-points
  - Maximum 2-3 levels

1. Numbered items for sequence
2. Each item starts with action verb
3. Complete sentences with periods
```

## Tables

### Structure
| Column 1 | Column 2 | Column 3 |
|----------|----------|----------|
| Data 1 | Data 2 | Data 3 |
| Data 4 | Data 5 | Data 6 |

### Best Practices
- Clear column headers
- Consistent alignment
- Avoid empty cells
- Use for comparisons, not prose
- Right-align numbers
- Left-align text

### When to Use Tables
- Comparing data
- Showing schedules
- Presenting specifications
- Financial information

---

## Numbers and Dates

### Numbers
- Spell out one through nine
- Use numerals for 10 and above
- Use commas for thousands (1,000)
- Use consistent decimal places
- Right-align in tables

### Dates
- Standard format: January 15, 2024
- Short format: Jan 15, 2024
- ISO format: 2024-01-15
- Be consistent within document

### Currency
- Include currency symbol: $1,500.00
- For large numbers: $1.5 million
- In tables, align by decimal

---

## Visual Elements

### Images
- High resolution (300 DPI for print)
- Include captions
- Reference in text ("as shown in Figure 1")
- Consider file size for digital

### Charts and Graphs
- Clear titles and labels
- Legend when needed
- Consistent colors
- Accessible (consider colorblind users)

### Diagrams
- Clear flow direction
- Consistent shapes for elements
- Include key/legend
- Keep simple and readable

## Professional Document Checklist

### Before Sending
- [ ] Spell check completed
- [ ] Grammar check completed
- [ ] All placeholders replaced
- [ ] Page numbers correct
- [ ] Table of contents updated
- [ ] Cross-references verified
- [ ] Images properly placed
- [ ] Format consistent throughout
- [ ] File named appropriately
- [ ] Sensitive data protected

### File Naming Convention
```
[Type]_[Title]_[Version]_[Date].[ext]

Examples:
Proposal_WebsiteRedesign_v2_2024-11-15.pdf
Contract_ServiceAgreement_v1_2024-11-15.docx
Report_QuarterlyStatus_v1_2024-11-15.pdf
```

## Document Security

### Confidentiality Levels
- **Public**: No restrictions
- **Internal**: Company employees only
- **Confidential**: Specific authorized personnel
- **Restricted**: Highest security, limited access

### Protection
- Password protect sensitive documents
- Watermark confidential documents
- Use secure file sharing
- Maintain version control

---

## Accessibility

### Text
- Use sufficient color contrast
- Don't rely on color alone
- Use readable font sizes
- Avoid all caps for body text

### Structure
- Use proper heading hierarchy
- Add alt text to images
- Use descriptive link text
- Ensure table headers are marked

### Testing
- Check with screen reader
- Verify keyboard navigation
- Test color contrast
- Validate with accessibility checker
