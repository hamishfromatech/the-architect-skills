# Report Writing Best Practices

## Report Structure Principles

### The Pyramid Principle
Start with the conclusion, then support it with evidence:
1. **Main conclusion** - The key takeaway
2. **Supporting arguments** - Key points that support the conclusion
3. **Evidence** - Data, facts, and details

### Executive Summary First
- Write the executive summary LAST
- Keep it to one page
- Include key findings and recommendations
- Should be readable standalone

---

## Writing Style Guidelines

### Clarity
- Use simple, direct language
- One idea per paragraph
- Avoid jargon unless audience-appropriate
- Define acronyms on first use

### Conciseness
- Remove unnecessary words
- Use bullet points for lists
- Aim for shorter sentences (15-20 words average)
- Cut paragraphs that don't add value

### Objectivity
- Present facts without bias
- Use neutral language
- Avoid emotional words
- Cite sources for claims

### Actionability
- Every report should have clear next steps
- Recommendations should be specific
- Assign owners to action items
- Include deadlines

---

## Report Types and Their Purpose

| Type | Purpose | Key Elements |
|------|---------|--------------|
| Status Report | Track progress | Completion %, blockers, next steps |
| Progress Report | Update milestones | Timeline, achievements, issues |
| Analysis Report | Present findings | Data, insights, recommendations |
| Executive Summary | High-level overview | Key points, decisions, next steps |
| Incident Report | Document issues | Timeline, impact, resolution |

## Data Presentation

### Tables
Use when:
- Comparing multiple values
- Showing precise numbers
- Presenting structured data

Best practices:
- Clear column headers
- Consistent number formatting
- Highlight important values
- Keep tables simple

### Charts
Use when:
- Showing trends over time
- Comparing proportions
- Visualizing relationships

Best practices:
- Choose right chart type
- Clear labels and legend
- Don't overload with data
- Use color meaningfully

## Common Report Mistakes

### 1. Missing the "So What?"
❌ "Sales increased 15%."
✅ "Sales increased 15%, exceeding our target by 5%. This improvement is primarily due to the new marketing campaign."

### 2. Information Overload
❌ Including every data point
✅ Summarizing key findings with supporting evidence

### 3. Unclear Recommendations
❌ "We should look into this further."
✅ "We recommend hiring 2 additional developers by Q2 to meet the project deadline."

### 4. Passive Voice Overuse
❌ "The issue was identified and resolved."
✅ "The team identified and resolved the issue."

### 5. Missing Context
❌ "Revenue: $1.5M"
✅ "Revenue: $1.5M (up 20% from last quarter, on target)"

## Status Report Best Practices

### Weekly Status Reports Should Include:
1. **Summary sentence** - Overall status in one line
2. **Accomplishments** - What was done
3. **In Progress** - What's being worked on
4. **Blockers** - What's preventing progress
5. **Next Steps** - What's planned

### Status Indicators
- 🟢 **On Track** - Meeting expectations
- 🟡 **At Risk** - Potential issues
- 🔴 **Off Track** - Significant problems

## Executive Summary Checklist

- [ ] Can be read in under 2 minutes
- [ ] Contains the main conclusion
- [ ] Lists key findings (3-5 max)
- [ ] Includes top recommendations
- [ ] Identifies decisions needed
- [ ] Written last, after full report
- [ ] Self-contained (understandable without full report)

---

## Action Item Format

Every action item should have:
1. **What** - Clear description
2. **Who** - Single owner
3. **When** - Specific deadline
4. **Why** - Context (if needed)

Example:
- ✅ "Complete API integration - @john_smith - Due: Nov 20"
- ❌ "John to do the API thing soon"

---

## Report Review Checklist

Before submitting:
- [ ] Executive summary is complete and accurate
- [ ] All data is current and accurate
- [ ] Recommendations are specific and actionable
- [ ] Action items have owners and deadlines
- [ ] Spelling and grammar checked
- [ ] Format is consistent throughout
- [ ] File naming convention followed
- [ ] Confidentiality level appropriate

## Formatting Standards

### Fonts
- Use professional fonts (Arial, Calibri, Times New Roman)
- Body: 11-12pt
- Headings: 14-16pt
- Titles: 18-24pt

### Spacing
- Line spacing: 1.15 or single
- Paragraph spacing: 6-12pt after
- Margins: 1 inch (standard)

### Headings
- Use consistent hierarchy
- Number sections for reference
- Don't skip heading levels

### Colors
- Use color sparingly and purposefully
- Ensure accessibility (contrast)
- Consider printing (black/white)

---

## Communication Tips

### Delivering Bad News
1. Be direct but empathetic
2. Provide context
3. Explain impact
4. Offer solutions
5. Outline next steps

### Requesting Action
1. State what's needed clearly
2. Explain why it's important
3. Provide deadline
4. Offer support if needed

### Presenting Options
1. Present all viable options
2. Include pros/cons for each
3. Provide recommendation
4. Explain reasoning
